I wrote a post the other day laying out what Id consider a minimum viable footprint for API operations.My vision of just exactly what an API is, has gone beyond just the technical, ever since I started API Evangelist back in July of 2010.Early on I saw this was more than just about the API endpoints, and documentation, code samples, and many other building blocks were essential to the success (or failure) of any API platform, area, or ecosystem.This recent post was an attempt, here in 2015, to quantify what I would consider to be a minimum definition for API operation.After writing it I wanted to take another stab at actually creating a portal, that would stand up to the API rhetoric that I regularly produce.
